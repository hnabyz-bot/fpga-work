## 상세 구현 계획

### 1단계: 환경 설정 스크립트 (setup_env.sh)

#### 구현 항목

- [ ] Questa 환경 자동 감지 및 로드
- [ ] 프로젝트 경로 자동 설정
- [ ] Vivado IP 경로 설정
- [ ] 환경 검증 기능

#### 검증 기준

- `source setup_env.sh` 실행 후 `vsim -version` 정상 출력
- 환경 변수 $BLUE_HD_ROOT가 올바른 경로를 가리킴
- vsim not found 에러가 발생하지 않음

#### 위험 요소 및 대응

| 위험 | 확률 | 영향 | 대응 방안 |
|------|------|------|----------|
| 경로 의존성 하드코딩 | 중간 | 높음 | 자동 경로 감지 추가 |
| 라이선스 파일 없음 | 낮음 | 높음 | 명확한 에러 메시지 제공 |

---

### 2단계: Makefile 빌드 시스템

#### 구현 항목

- [ ] work 라이브러리 자동 생성
- [ ] 소스 컴파일 순서 최적화
- [ ] GUI/Batch 모드 지원
- [ ] 환경 검증 타겟 (check_env)

#### 컴파일 순서

```
1. 파라미터 정의 (p_define.sv)
2. IP 시뮬레이션 모델 (선택적)
3. TI-ROIC 인터페이스 모듈
4. 공통 모듈 (FIFO, SPI, I2C 등)
5. MIPI CSI-2 모듈 (선택적)
6. 최상위 모듈 (blue_hd_top.sv)
7. 테스트벤치 지원 모듈
8. 테스트벤치
```

#### 검증 기준

- `make compile` 실행 시 오류 없이 완료
- `make sim`으로 GUI가 정상적으로 열림
- `make sim_batch`로 배치 실행이 완료되고 종료 코드 0 반환

---

### 3단계: 단일 테스트 실행 스크립트 (run_test.sh)

#### 구현 항목

- [ ] 테스트벤치 이름 검증
- [ ] GUI/Batch/Compile 모드 지원
- [ ] 자동 소스 컴파일
- [ ] 사용법 도움말

#### 워크플로우

```
1. 인자 파싱 (testbench_name, mode)
2. 환경 설정 스크립트 로드
3. 테스트벤치 존재 확인
4. 소스 컴파일 (compile 함수)
5. 시뮬레이션 실행 (run_simulation 함수)
6. 결과 보고
```

#### 검증 기준

- `./run_test.sh test_bench gui`로 GUI 정상 실행
- `./run_test.sh sequencer_fsm_tb batch`로 배치 완료
- 존재하지 않는 테스트벤치 실행 시 에러 메시지 표시

---

### 4단계: 회귀 테스트 스크립트 (run_regression.sh)

#### 구현 항목

- [ ] 빠른/전체 테스트 모드 지원
- [ ] 타임스탬프별 결과 저장
- [ ] 패스/패일 자동 판정
- [ ] 텍스트 리포트 생성
- [ ] 종료 코드 반환 (성공: 0, 실패: 1)

#### 워크플로우

```
1. 인자 파싱 (--all, --quick, --verbose)
2. 결과 디렉토리 생성 (타임스탬프)
3. 테스트 목록 선택
4. 순차적 테스트 실행
5. 결과 수집 및 판정
6. 리포트 생성
7. 종료 코드 반환
```

#### 검증 기준

- `./run_regression.sh --quick`으로 3개 테스트 실행
- `./run_regression.sh --all`으로 9개 테스트 실행
- regression_results/YYYYMMDD_HHMMSS/ 디렉토리에 결과 저장
- regression_report.txt에 패스/패일 요약 표시
